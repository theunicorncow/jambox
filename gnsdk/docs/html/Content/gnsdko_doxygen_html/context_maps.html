GNSDK C API Reference

GNSDK Data Model

The GNSDK data model represents Gracenote media elements and metadata. The model establishes interrelationships among Gracenote Data Objects (GDOs) and the metadata they contain or reference. The first table below maps GDO types to their corresponding Child GDOs and values.

Tips on Using the Data Model Table

The Parent GDO Types and Child and Value Keys in the first two columns link directly to their corresponding definitions in the GNSDK C API reference. The GDO Child Keys in the third column instead link to their corresponding item in the first column. of the table.

In addition to these tables, there is a supporting data model document called the "GNSDK Data Dictionary. This document describes common GDOs and metadata values.

Locale-Dependent Values and GDO Values in Partial Results

See Locale-Dependent Values, for locale-dependent (list-dependent) metadata values like Genre, artist Origin, artist Era, and artist Type (collectively known as GOET values), as well as mood and tempo, contributor lists, roles, and others.

Gracenote sometimes returns partial results in a GDO. For a list of values returned in partial results see GDO Partial Result Values For information about working with GDO partial and full results, see "Gracenote Data Objects (GDOs)" in the GNSDK Developer's Guide.

 

Important: Unlike other GDO Keys, GNSDK_GDO_CHILD_MATCH does not return its own GDO type. Instead it returns one of GNSDK_GDO_TYPE_ALBUM or GNSDK_GDO_TYPE_CONTRIBUTOR, depending on what was matched.
