gnsdk_error_t GNSDK_API gnsdk_playlist_generate_morelikethis ( gnsdk_user_handle_t  user_handle,
gnsdk_playlist_collection_handle_t  h_collection,
gnsdk_gdo_handle_t  h_gdo_seed,
gnsdk_playlist_results_handle_t *  ph_results 
)

Generates a Playlist using the Gracenote "More Like This" algorithm and GDO Seed input. The algorithm matches tracks primarily on Mood and Genre attributes.

Parameters
user_handle[in] Handle to a user for the user requesting the "More Like This" playlist
h_collection[in] Handle to a Playlist collection summary
h_gdo_seed[in] Handle to a GDO Seed (required)
ph_results[out] Pointer to receive generated Playlist results handle
Returns
PLERR_NotInited gnsdk_playlist_initialize was not successfully initiated
PLERR_InvalidArg Given ph_results is null
PLERR_NoError Playlist was successfully generated

Important:

Playlist generates playlists from a single collection summary at a time. It does not support the derivation of playlists across multiple collection summaries. Applications that require deriving a playlist from multiple collection summaries must create a master collection summary that spans the required collections.

gnsdk_error_t GNSDK_API gnsdk_playlist_generate_playlist ( gnsdk_user_handle_t  user_handle,
gnsdk_cstr_t  pdl_statement,
gnsdk_playlist_collection_handle_t  h_collection,
gnsdk_gdo_handle_t  h_gdo_seed,
gnsdk_playlist_results_handle_t *  ph_results 
)

Generates a Playlist. Use this function to generate Playlist results. As a best practice and for processing efficiency, we recommend calling gnsdk_playlist_statement_validate() before generating results.

Parameters
user_handle[in] Handle to a user for the user requesting the playlist
pdl_statement[in] PDL statement string
h_collection[in] Handle to a Playlist collection summary
h_gdo_seed[in] Handle to a GDO Seed (if necessary)
ph_results[out] Pointer to receive generated Playlist results handle
Returns
PLERR_NotInited gnsdk_playlist_initialize was not successfully initiated
PLERR_InvalidArg Given pdl_statement is empty, or given ph_results is null
PLERR_StatementError PDL statement can't be validated, due to missing or incorrect information. If PLERR_StatementError is returned, details on problems with the PDL statement can be shown by calling gnsdk_manager_error_extended_info.
PLERR_NoError Playlist was successfully generated

Note:

Streamline your Playlist implementation by using the gnsdk_playlist_generate_morelikethis() function, which uses the "More Like This" algorithm to generate optimal playlist results and eliminates the need to create and validate Playlist Definition Language statements.

Important:

Playlist generates playlists from a single collection summary at a time. It does not support the derivation of playlists across multiple collection summaries. Applications that require deriving a playlist from multiple collection summaries must create a master collection summary that spans the required collections.

gnsdk_error_t GNSDK_API gnsdk_playlist_statement_validate ( gnsdk_cstr_t  pdl_statement,
gnsdk_playlist_collection_handle_t  h_collection,
gnsdk_bool_t *  pb_seed_required 
)

Validates a Playlist Definition Language (PDL) statement. Use this function to determine: If the PDL is invalid, such as using an attribute name that does not exist in the collection or using an invalid keyword for an operator. If the PDL requires a Seed to generate results. As a best practice, we recommend calling this function to validate PDL statements especially unknown statements before calling gnsdk_playlist_generate_playlist().

Parameters
pdl_statement[in] PDL statement string
h_collection[in] Handle to a Playlist collection summary
pb_seed_required[out] Pointer to receive a boolean value indicating if a Playlist Seed is required
Returns
PLERR_NotInited gnsdk_playlist_initialize was not successfully initiated
PLERR_StatementError PDL statement can't be validated, due to missing or incorrect information. If PLERR_StatementError is returned, details on problems with the PDL statement can be shown by calling gnsdk_manager_error_extended_info.
PLERR_InvalidArg Given pdl_statement is empty or invalid
PLERR_NoError PDL statement was successfully validated
