Header Files and Libraries

GNSDK consists of a set of shared modules. The GNSDK Manager module is required for all applications. All other modules are optional. Your application's feature requirements determine which of the optional modules should be used.

As shown in the following table, each GNSDK module has one or more corresponding header files. Other header files are also provided for public types and error codes.

Note: The libraries can be linked in any order for all modules, except for the GNSDK Manager library. The GNSDK Manager library must be last in the linking order.

Module

Header Files

Libraries

General

Required.

gnsdk.h. This header file also includes these required header files:

  • gnsdk_defines.h: GNSDK types
  • gnsdk_platform.h: GNSDK platform specific types (one header for each supported platform)
  • gnsdk_errors.h: GNSDK error macros
  • gnsdk_error_codes.h: GNSDK error definitions
  • gnsdk_version.h: GNSDK current version definition (this header is optional)

 

GNSDK Manager

Required. These header files provide core functionality necessary to all Gracenote modules.

  • gnsdk_manager.h
  • gnsdk_manager_gdo.h
  • gnsdk_manager_lists.h
  • gnsdk_manager_list_values.h
  • gnsdk_manager_locales.h
  • gnsdk_manager_logging
  • libgnsdk_manager.so
  • libgnsdk_manager.dylib
  • gnsdk_manager.dll

MusicID

Optional. This header file provides CD TOC, text, and fingerprint identification for music.

  • gnsdk_musicid.h
  • libgnsdk_musicid.so
  • libgnsdk_musicid.dylib
  • gnsdk_musicid.dll

MusicID-File

Optional. Provides music file-based identification using LibraryID, AlbumID, and TrackID processing functionality.

  • gnsdk_musicid_file.h
  • libgnsdk_musicid_file.so
  • libgnsdk_musicid_file.dylib
  • gnsdk_musicid_file.dll

DSP

Optional. Provides digital signal processing functionality required for fingerprint generation.

  • gnsdk_dsp.h
  • libgnsdk_dsp.so
  • libgnsdk_dsp.dylib
  • gnsdk_dsp.dll

Link

Optional. Provides enriched metadata (non-core content and third party metadata.).

  • gnsdk_link.h
  • libgnsdk_link.so
  • libgnsdk_link.dylib
  • gnsdk_link.dll

MoodGrid

Optional. A feature of Playlist to visually group content on a grid based on mood metadata.

  • gnsdk_moodgrid.h
  • libgnsdk_moodgrid.so
  • libgnsdk_moodgrid.dylib
  • gnsdk_moodgrid.dll

Playlist

Optional. Provides functionality to generate and manage playlists and playlist collections.

  • gnsdk_playlist.h
  • libgnsdk_playlist.so
  • libgnsdk_playlist.dylib
  • gnsdk_playlist.dll

Local

Database

Optional. Used for embedded system applications that require a local database that contains pre-defined Gracenote metadata.

  • gnsdk_lookup_local.h
  • libgnsdk_lookup_local.so
  • libgnsdk_lookup_local.dylib
  • gnsdk_lookup_local.dll

SQLite

Optional. Provides local caching implementation based on SQLite, and provides the default storage implementation to support local databases.

  • gnsdk_sqlite.h
  • libgnsdk_sqlite.so
  • libgnsdk_sqlite.dylib
  • gnsdk_sqlite.dll
